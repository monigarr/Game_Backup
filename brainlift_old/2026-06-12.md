## 3. Challenges Faced

- **Broadcast scope vs sovereignty**: PRD S9 and Appendix H specify read-only, user-initiated, no device control, opt-in. Had to ensure UDP beacon carries only metadata/QR (no audio) and respects CreditsWallet spend gate.
- **Narrative 26-module consistency**: Needed to ensure every conjugation unlocks a unique lore fragment without duplication; RAG retrieval must always attribute sources.
- **Analytics without real Firebase SDK**: LocalStubBackend must emulate the event shape so evals can consume it; future Firebase drop-in must be zero-code-change.
- **WebGL mic + NGO constraints**: Browser autoplay/mic policy + NGO WebGL transport limits required explicit fallback note in SETUP.
- **Time pressure on Final day**: D5 had to land all code + seams so D6 (Sat) could focus on video recording, final verification, and submission packaging.

---

## 4. Solutions Found

- **Broadcast**: Implemented `BroadcastService` with `BroadcastMode` enum (Off / LAN_UDP / Simulated), `TryBroadcast(ConjugationId)` that checks credits, emits UDP on port 47777 with JSON {module, phrase, timestamp, qr_payload}, and falls back silently if no network. Added `BroadcastHubScreen` UI hook.
- **Narrative**: Extended `NarrativeFragment` ScriptableObject with `requiresConjugation`, `loreText`, `culturalSource`, `ragQuery`. `NarrativeService` now listens to `ProgressionService.OnModuleMastered` and appends RAG-retrieved fragments to `NarrativeLogScreen` with source attribution.
- **Analytics**: Added `LogEvent(string name, Dictionary<string,object> props)` to `LocalStubBackend`; called from key services. Events written to `evals/reports/analytics.json` for Final eval run.
- **WebGL**: Confirmed jslib works; added explicit "WebGL hosted URL" section in SETUP.md with itch.io or Firebase Hosting steps; workflow builds Standalone + WebGL.
- **Docs seeds**: Added extraction notes as "future seams" so the architecture is explicitly a reusable software factory (per PRD G-010 / Roadmap).

## 5. Pivots / Decisions

- **Decision**: Keep Broadcast LAN_UDP as UDP multicast beacon (no pairing) — simplest, no external deps, satisfies "ambient device broadcasting" viral hook without real IoT control (which was explicitly non-goal).
- **Decision**: Narrative expansion stays within sovereign corpus + RAG; no generative Kanien'kéha phrases (per AI-004 guardrails).
- **Decision**: Analytics via stub for Final; real Firebase left as post-submission seam (ClientRequirements does not require live backend).
- **Decision**: WebGL remains secondary target (Desktop primary for voice latency); docs clearly state the hosted URL requirement for Deliverable 1 Final gate.
