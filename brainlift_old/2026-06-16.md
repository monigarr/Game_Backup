## 3. Challenges Faced

- **Compile time and iteration friction in Unity**: Long build cycles + Unity AI Assist behavior magnified every scope miscalculation and slowed learning velocity to a crawl.
- **Hardware limitations**: Laptop (documented baseline) could not sustain reliable Unity 3D + multiplayer + audio/video workloads within the aggressive 1-day MVP timeline.
- **Unreliable initial decision**: The original scope (full 26-module narrative, voice, 3D hub, NGO multiplayer, on-device sovereignty) was too large for the available time, equipment, and team capacity; this was not recognized early enough.
- **Fail grade impact**: Submission did not demonstrate the required working multiplayer experience or polish, triggering the need for a total reset rather than incremental fixes.
- **Emotional / process cost of abandonment**: Walking away from weeks of prior work and the linked GitHub repo (karonhiake) required a clear decision framework and documentation to turn the failure into documented learning.

---

## 4. Solutions Found

- **Complete pivot decision**: After the fail grade, chose to start fresh with Path 2 (Phaser 3 2D + Socket.io) rather than attempting to salvage the Unity project. This allowed rapid iteration, 60 fps browser gameplay, and immediate multiplayer rooms without heavy build pipelines.
- **New lightweight stack selection**: Phaser 3 + Vite + Socket.io + geometric visuals eliminated asset and compile bottlenecks while still delivering real-time multiplayer, progression (levels, unlocks, arenas), hazards, dash ability, and all required deliverables.
- **Documentation of the pivot**: Created this dedicated brainlift entry + updated the new project's BRAINLIFT.md with an explicit "backup game" note so the failure-to-success arc is preserved for evaluators.
- **Scope reset to MVP**: The new OrbChase game was deliberately scoped to exactly what could be built and deployed in one day on the current hardware, satisfying every ClientRequirements item without overreach.
